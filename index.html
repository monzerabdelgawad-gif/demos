⚙ Endpoint Map — POST Data from n8n
01 — Post Daily Stats
POST
window.location (same file — JS handles it)
n8n calls postDailyStats(payload) via the injected script, or you use an HTTP Request node to POST JSON to the dashboard URL with type="daily_stats".

02 — Post Monthly Stats
POST
type: "monthly_stats"
Triggers monthly dashboard update. Call this once per day from n8n. Includes top 10 questions, 5 winning products, and low performers.
{
  "type": "monthly_stats",
  "messages_received": 6420,
  "messages_sent": 5980,
  "orders": 412,
  "top_questions": [
    { "q": "Question text", "count": 320 }
  ],
  "top_products": [
    {
      "name": "Product A",
      "inquiries": 240,
      "orders": 88
    }
  ],
  "low_inquiry_products": [
    { "name": "Slow Product", "inquiries": 3 }
  ],
  "low_order_products": [
    { "name": "Zero Convert", "orders": 0 }
  ]
}
n8n Setup — HTTP Request Node
1. In n8n, use HTTP Request node → Method: POST
2. URL: the URL where this dashboard file is hosted (e.g. on your server or GitHub Pages)
3. Body Content Type: JSON
4. Build the JSON payload using n8n expressions from your Google Sheet or workflow data
5. The dashboard listens for window.addEventListener('message', ...) and also exposes window.injectData(payload) for direct JS injection
6. For hosted deployments, the dashboard server should accept POST to /api/data and push via WebSocket or SSE — or use localStorage as a local cache
